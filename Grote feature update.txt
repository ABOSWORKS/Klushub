Dit is een kritiek onderdeel om in te richten. De beste oplossing voor jouw setup (GitHub Pages + Supabase) is:

Aanbevolen aanpak: Supabase Edge Function + Resend

Resend is een e-mailservice met een gratis tier van 3.000 e-mails per maand. Een Supabase Edge Function wordt automatisch getriggerd zodra er een klus in de database wordt opgeslagen.

Stap 1 — Resend account aanmaken

Ga naar resend.com → maak een gratis account → ga naar API Keys → maak een nieuwe key aan → kopieer de key.

Stap 2 — Edge Function aanmaken in Supabase

Ga naar Supabase → Edge Functions → New Function → noem het stuur-beheer-email.

Plak deze code:

Stap 5 — Eigen domein (optioneel maar aanbevolen)

Resend vereist een geverifieerd domein om e-mails te versturen vanuit @klushub.nl. Ga naar Resend → Domains → voeg klushub.nl toe → voeg de DNS records toe bij je domeinbeheerder. Zonder eigen domein kun je tijdelijk sturen vanuit @resend.dev.

Dit is de meest robuuste aanpak — volledig serverless, werkt automatisch bij elke nieuwe klus en kost niets bij je huidige volume.
