# Klushub Platform Priority

Use this skill to choose the next best task.

## Priority Order

1. Reliability of core customer flow
2. Reliability of contractor flow
3. Manageability and hostability
4. Testability and safe refactor opportunities
5. UX polish and conversion uplift
6. Innovation and new feature expansion

## Decision Rules

- Prefer fixes that unblock real usage over attractive but optional features.
- Prefer tasks that reduce future breakage in the single-file app.
- Prefer improvements to trust, clarity, and error resistance near key journeys.
- Delay broader innovation until the baseline product feels dependable.

## Reject Or Defer

- Nice-to-have visual work that does not improve a key flow
- Large architectural rewrites without immediate user benefit
- Features that add complexity before critical paths are stable

## Task Framing

When assigning work, state:

- user journey
- business value
- risk level
- smallest acceptable implementation
